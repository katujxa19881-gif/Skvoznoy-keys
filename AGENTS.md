# Правила работы — этап подготовки

Сейчас этап «Постановка проблемы».
Собираем информацию о продукте в файлы
папки 0-подготовка/.

## Универсальные правила (этапы 0–6 подготовки)

1. Никогда не сохраняй и не редактируй файлы
   проекта без явного «Ок» от пользователя.
   Сначала покажи результат в чате — обсудим.
   После «Ок» — записывай в файл.

2. Структура папки фиксирована (см. ниже).
   Имена файлов и пути не меняй.

3. Если что-то нужно добавить вне структуры —
   спроси меня перед созданием.

## Структура (этап 0)

0-подготовка/
├── проблема.md          — финальная формулировка проблемы
├── аудитория.md         — целевая аудитория
├── портрет.md           — портрет пользователя
├── jtbd.md              — Jobs To Be Done
└── интервью/
    ├── гайд-интервью.md
    ├── транскрипты/     — юзер кладёт сюда транскрипты сам
    ├── анализ/          — анализ по каждому интервью
    │   └── анализ-[имя]-[дата].md
    └── итог-всех-интервью.md   — сводный анализ

0-бриф.md — главный итоговый файл этапа 0
            (сборка всего вышеуказанного).

## Дальнейшие этапы

На этапах 1–6 этот файл будет дополняться
новыми разделами «Структура» (новые файлы
добавятся к списку выше). Универсальные
правила про «Ок» и фиксированные пути —
остаются неизменными.

Перед этапом разработки (этап 9) этот файл
будет полностью переписан — там появятся
правила для написания кода (план, коммиты,
логи, автопроверка).
